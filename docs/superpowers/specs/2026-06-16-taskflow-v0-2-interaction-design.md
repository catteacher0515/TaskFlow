# TaskFlow v0.2 交互修正规格

日期：2026-06-16

## 背景

当前版本已经能创建项目、展示模板、推进「每周 GitHub 精选」候选仓库，并记录反馈。但实际使用后暴露了三个问题：

1. 同一个候选仓库可以反复点击状态按钮，导致反馈数量和最近反馈失真。
2. 通用任务只能开始和完成，不能拆成小任务推进。
3. 当前面板只显示最近 3 条反馈，没有入口查看完整反馈历史。

v0.2 的目标是修正这三个问题，让反馈代表真实推进，让通用任务可以拆解，让历史反馈可追溯。

## 范围

v0.2 包含：

- 候选仓库和通用任务的反馈去重规则。
- 通用任务的最多三层任务树。
- 小任务的完成反馈和熵减反馈。
- 父任务提前完成与未处理子任务归档规则。
- 独立反馈页。
- 当前面板的「查看全部反馈」入口。

v0.2 不包含：

- 无限层级嵌套。
- AI 推荐当前应该做哪个任务。
- 完整模板编辑器。
- 自动周期任务生成。
- 复杂统计图表。
- 普通操作审计日志。

## 反馈规则

反馈日志只记录有效推进，不记录普通点击和普通编辑。

### 候选仓库反馈

「每周 GitHub 精选」的候选仓库最多贡献一次小反馈。

候选仓库第一次进入以下任一终局状态时，生成小反馈：

- 入选。
- 备选。
- 淘汰。

候选仓库进入以下状态时，不生成反馈：

- 未测。
- 测试中。

如果候选仓库已经贡献过小反馈，后续状态修订只更新当前状态，不再生成新的反馈。例如：

- 备选改成入选：更新状态，不新增反馈。
- 入选改成淘汰：更新状态，不新增反馈。
- 重复点击当前状态：无效操作，不新增反馈。

前端不应把当前状态显示为可点击按钮。后端也必须保证同一候选仓库不会因为重复请求贡献多次反馈。

### 通用任务反馈

通用任务的小任务最多贡献一次小反馈。

小任务第一次进入以下任一终局状态时，生成小反馈：

- 完成。
- 不做了。

「不做了」是熵减反馈。它表示用户主动减少任务负担，而不是失败。

同一个小任务后续在终局状态之间切换，只更新状态，不再新增反馈。

## 通用任务模型

通用任务不使用固定阶段。通用任务使用最多三层任务树：

```text
大任务
-> 小任务
-> 检查项 / 更小任务
```

规则：

- 最大深度为三层。
- 不要求每个任务都有子任务。
- 不要求创建任务时一次性拆完。
- 用户可以边做边添加小任务。
- 任务拆解不应被设计成越细越好；工具应该保留随机性空间。

### 任务状态

通用任务节点支持以下状态：

- 未开始。
- 进行中。
- 完成。
- 不做了。
- 未处理。

「未处理」只用于父任务完成后，跟随父任务一起归档的未完成子任务。它不产生反馈。

### 父任务进度

父任务进度由子任务自动汇总。例如：

```text
整理播客选题 2 / 5
```

父任务不需要用户手动维护进度条。

### 父任务完成

父任务可以在子任务未全部处理时完成。

完成父任务前，界面提示未处理数量，例如：

```text
还有 3 个未处理的小任务
```

该提示不阻止完成。父任务完成后：

- 未完成子任务跟随父任务归档。
- 这些子任务显示为「未处理」。
- 不自动算完成。
- 不产生反馈。

父任务完成本身生成大反馈。

## 独立反馈页

顶部导航新增「反馈」入口。

当前面板继续显示最近 3 条反馈，并提供「查看全部反馈」入口。点击后进入独立反馈页。

反馈页按时间倒序显示完整反馈历史。每条反馈至少显示：

- 时间。
- 所属项目。
- 反馈大小：小反馈或大反馈。
- 反馈类型：完成、熵减、槽位填充、父任务完成、项目完成等。
- 反馈内容。

反馈页第一版支持以下筛选：

- 全部。
- 小反馈。
- 大反馈。
- 当前项目。

反馈页展示的是反馈日志，不是所有操作日志。普通状态修订、标题编辑、URL 编辑等不在反馈页展示。

## 页面行为

### 每周 GitHub 精选详情

候选仓库卡片中：

- 当前状态显示为状态标签。
- 可点击按钮只显示其他可切换状态。
- 如果候选已经贡献过小反馈，终局状态切换不再新增反馈。
- 如果候选未贡献过小反馈，第一次进入入选、备选或淘汰时新增小反馈。

### 通用任务详情

通用任务详情中：

- 显示任务树。
- 可以在未达到最大深度的任务节点下添加子任务。
- 可以将任务标记为进行中、完成或不做了。
- 完成和不做了都只贡献一次小反馈。
- 父任务可以提前完成。
- 父任务完成后，未处理子任务显示为未处理并归档。

### 当前面板

当前面板中：

- 反馈数量表示有效反馈总数，不是点击次数。
- 最近反馈只显示最后 3 条。
- 提供「查看全部反馈」入口。

## 验收标准

- 同一个候选仓库反复切换终局状态时，反馈总数最多增加一次。
- 当前状态不再以可重复点击按钮出现。
- 通用任务可以添加最多三层任务树。
- 通用任务小任务可以标记完成或不做了，同一个小任务最多贡献一次反馈。
- 父任务可以在子任务未全部处理时完成。
- 父任务完成后，未处理子任务显示为未处理且不产生反馈。
- 顶部导航出现「反馈」。
- 当前面板最近反馈区域出现「查看全部反馈」。
- 反馈页可以查看超过 3 条的完整反馈历史。
- 反馈页可以按全部、小反馈、大反馈、当前项目筛选。

## 测试要求

- 共享逻辑测试覆盖候选仓库反馈去重。
- 共享逻辑测试覆盖通用任务最多三层限制。
- 共享逻辑测试覆盖完成、不做了、未处理三种终局行为。
- 服务端路由测试覆盖反馈去重和任务树写入。
- 客户端测试覆盖候选状态按钮不显示当前状态。
- 客户端测试覆盖通用任务添加子任务、完成任务、不做了。
- 客户端测试覆盖反馈页和查看全部入口。
