# 任务进度可视化工具设计

日期：2026-06-15

## 背景

用户当前的问题不是简单缺少 Todo，而是同时开启了太多任务，许多任务都有进度但没有完成，导致缺少正反馈。典型场景是「每周 GitHub 精选」：用户需要亲测多个 GitHub 仓库，最终选出 5 个值得推荐的项目。许多候选仓库会被淘汰，但“排除不合适选项”过去没有被用户感知为有效产出。

第一版工具要解决三个目标，优先级如下：

1. 提供完成感和正反馈。
2. 让进度一眼可见。
3. 限制同时进行的任务数量，并在失控时强制收束。

## 产品定位

第一版是一个模板优先的本地任务进度 Web App。它不是通用 Todo 清单，而是服务于探索性强、周期性强、容易发散、需要持续推进反馈的任务。

第一版必须能真实支撑下一期「每周 GitHub 精选」完整跑完，同时保留复用到其他周期任务的结构。

## 范围

第一版包含：

- 普通项目创建。
- 从模板创建项目。
- 模板定义阶段、槽位、推进对象、状态流、最小推进动作、重复周期和预警规则。
- 内置「每周 GitHub 精选」模板。
- 小反馈：处理一个推进对象并得到明确结论。
- 大反馈：槽位填充、阶段完成、项目接近交付。
- 三类预警：并行过多、截止风险、停滞太久。
- 强约束的收束模式。
- 数据保存到项目目录中的本地文件。

第一版不包含：

- 登录和账号系统。
- 云同步。
- 多设备协作。
- AI 自动推荐当前任务。
- 自动日历调度。
- 自动生成新一期周期任务。
- SQLite 或其他数据库。
- 复杂统计和复盘报表。

## 核心概念

### 模板

模板是一类可复用任务的蓝图。模板定义：

- 阶段。
- 槽位。
- 推进对象类型。
- 推进对象状态流。
- 哪些状态变化算一次小反馈。
- 最小推进动作。
- 重复规则。
- 预警规则。

模板修改只影响之后创建的新项目，不自动修改已经存在的旧项目。

### 项目

项目是一次具体任务实例，例如「每周 GitHub 精选 2026-W25」。项目可以从模板生成，也可以手动创建。

从模板创建项目时，项目保存一份模板结构快照。这样后续修改模板不会破坏旧项目的阶段、槽位或状态定义。

### 推进对象

推进对象表示一个项目中可以被逐个处理、逐个产生小反馈的对象。

全局系统不固定推进对象的名称和状态。不同模板自行定义推进对象。例如：

- 「每周 GitHub 精选」的推进对象是候选仓库，状态可以是未测、测试中、入选、备选、淘汰。
- 「文章写作」的推进对象可以是段落、小节或素材卡，状态可以是待写、草稿、待修改、完成。
- 「产品调研」的推进对象可以是竞品、用户反馈或问题假设，状态可以是待看、已记录、已归类、已放弃。

### 小反馈

小反馈用于告诉用户：刚刚这一步没有白做。

小反馈通常由推进对象产生。只要推进对象从未处理或处理中状态进入某个有明确结论的状态，就生成一条推进记录。对于「每周 GitHub 精选」，候选仓库被淘汰也算小反馈，因为它排除了一个不合适选项。

### 大反馈

大反馈用于告诉用户：项目正在接近最终交付。

大反馈通常来自：

- 槽位填充，例如推荐仓库 `3 / 5`。
- 阶段完成，例如亲测完成、初稿完成、发布完成。
- 项目交付状态变化。

### 收束模式

当系统检测到用户已经进入失控状态时，进入收束模式。收束模式的目标不是提醒更多事情，而是强制做减法。

收束模式下：

- 不能新建进行中项目。
- 不能把其他项目切到进行中。
- 用户必须手动选择一个当前唯一推进任务。
- 系统只显示客观风险信息，不替用户判断任务难度。
- 选定唯一推进任务后，用户需要选择或填写一个明确的小推进动作。
- 默认启动块是 5 分钟。5 分钟结束后，用户可以记录结果、继续推进或标记阻塞。

## 主要页面

### 当前面板

显示：

- 当前进行中的项目。
- 是否触发预警。
- 当前唯一推进任务。
- 下一个 5 分钟启动动作。
- 最近的小反馈和大反馈。

### 项目列表

按状态展示项目：

- 进行中。
- 待开始。
- 暂停。
- 已完成。

项目列表显示并行数量上限，例如 `进行中 4 / 3`。超过上限时触发预警并进入收束模式。

### 项目详情

显示一个项目的完整状态：

- 阶段进度。
- 推进对象列表。
- 槽位进度。
- 推进记录。
- 截止时间。
- 停滞天数。
- 当前可执行的最小推进动作。

### 模板管理

用于创建和编辑模板：

- 模板名称。
- 阶段配置。
- 推进对象类型。
- 推进对象状态流。
- 小反馈触发规则。
- 槽位配置。
- 最小推进动作。
- 重复规则。
- 预警规则。

### 收束模式

当触发预警时出现。它列出有风险的进行中项目，并展示客观信息：

- 截止时间。
- 停滞天数。
- 当前阶段。
- 槽位或关键进度。
- 是否超过并行上限。

用户手动选择一个当前唯一推进任务。系统不自动推荐，也不强制指定。

## 关键流程

### 从模板创建项目

用户选择模板，填写本期项目标题、截止时间和重复设置。系统生成项目实例，并复制模板结构快照。

重复规则第一版支持：

- 不重复。
- 每天。
- 每周。
- 每月。
- 工作日。
- 自定义间隔。

第一版记录重复规则，但不自动生成新一期。新一期需要用户手动确认生成，避免系统自动制造任务压力。

### 记录推进对象

用户在项目详情中添加或处理推进对象。每个模板定义自己的状态流和小反馈规则。

当推进对象进入有明确结论的状态时，系统生成推进记录，并显示为小反馈。

### 推进槽位和阶段

槽位和阶段由模板定义。不是所有项目都必须有槽位，也不是所有项目都有候选池。

如果项目有槽位，槽位填充会形成大反馈。如果项目有阶段，阶段完成会形成大反馈。

### 预警与收束

第一版只做三类预警：

1. 并行过多：进行中项目数量超过全局或模板设置的上限。
2. 截止风险：项目接近截止时间，但关键进度不足。
3. 停滞太久：项目或阶段连续 N 天没有新增推进记录。

触发预警后进入收束模式。用户必须手动选择一个当前唯一推进任务，并完成或记录一次明确推进动作，才能解除或重新评估预警。

## 内置模板：每周 GitHub 精选

该模板用于验证第一版体验。

默认阶段：

1. 候选收集。
2. 亲测。
3. 推荐选择。
4. 推荐理由写作。
5. 成稿。
6. 发布。

推进对象：

- 名称：候选仓库。
- 字段：仓库名、URL、来源、备注、测试记录、结论理由。
- 默认状态：未测、测试中、入选、备选、淘汰。
- 小反馈规则：候选仓库进入入选、备选或淘汰状态时生成推进记录。

槽位：

- 5 个推荐仓库槽位。
- 入选候选仓库可以填入槽位。
- 槽位进度显示为 `已填 / 5`。

默认最小推进动作：

- 亲测 1 个候选仓库。
- 给 1 个候选仓库写结论理由。
- 填入 1 个推荐槽位。
- 写完 1 个推荐理由草稿。

默认预警：

- 并行过多：使用全局进行中项目上限。
- 截止风险：临近截止时间但 5 个槽位未填满，或写作阶段尚未开始。
- 停滞太久：连续 N 天没有新增候选结论或写作推进记录。

## 技术架构

第一版采用本地 Web App 架构。

组成：

- 前端：浏览器界面，负责进度看板、项目详情、模板配置和收束模式。
- 本地服务：运行在用户电脑上，提供 API，负责读写本地数据文件。
- 数据文件：保存在项目目录下。

浏览器不能稳定读写项目目录中的本地文件，因此第一版不做纯静态网页。

## 本地数据文件

建议文件结构：

```text
data/
  settings.json
  templates/
    weekly-github-picks.json
  projects/
    <project-id>.json
  activity-log.jsonl
```

文件职责：

- `data/settings.json`：全局设置，例如同时进行项目上限、默认停滞天数。
- `data/templates/*.json`：模板定义。
- `data/projects/*.json`：项目实例与进度。
- `data/activity-log.jsonl`：追加式推进记录，用于后续复盘和统计。

数据流：

1. 用户在页面操作。
2. 前端调用本地 API。
3. 本地服务校验输入。
4. 本地服务写入 JSON 或 JSONL 文件。
5. 前端刷新状态、进度和预警。

## 异常处理

### 数据文件损坏

本地服务启动时校验 JSON。发现格式错误时：

- 不覆盖原文件。
- 提示具体文件路径。
- 尽可能给出错误位置。
- 保留原文件，允许用户修复或恢复备份。

### 模板修改

模板修改只影响新项目。旧项目使用创建时保存的模板结构快照。

### 重复任务堆积

第一版不自动生成周期任务。系统可以提示有新一期可创建，但需要用户手动确认。

### 收束模式绕过

进入收束模式后，前端和 API 都应阻止：

- 新建进行中项目。
- 将非唯一推进任务切换为进行中。
- 给非唯一推进任务记录新的推进动作。

## 验收标准

第一版完成时应满足：

- 能创建模板。
- 能从模板生成项目。
- 能配置「每周 GitHub 精选」模板，包括阶段、5 个推荐槽位、候选仓库状态流和最小推进动作。
- 能记录候选仓库入选、备选或淘汰，并生成小反馈。
- 能看到项目大进度，包括阶段进度、槽位进度和距离截止时间。
- 能触发并行过多预警。
- 能触发截止风险预警。
- 能触发停滞太久预警。
- 触发预警后能进入收束模式。
- 收束模式下只能选择并推进一个任务。
- 能启动 5 分钟启动块，并在结束后记录结果、继续或标记阻塞。
- 数据实际写入项目目录下的本地文件。
